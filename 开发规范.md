## 二、命名规范

### 1. 文件命名

- 

  **小驼峰** (lowerCamelCase)

- 

  示例: `user_profile_page.dart`, `auth_service.dart`

### 2. 类名/枚举/扩展名

- 

  **大驼峰** (UpperCamelCase / PascalCase)

- 

  示例: `UserProfilePage`, `AuthStatus`, `StringExtension`

### 3. 变量/函数/方法名

- 

  **小驼峰** (lowerCamelCase)

- 

  示例: `userName`, `fetchUserData()`, `handleLogin()`

### 4. 常量名

- 

  **全大写 + 下划线分隔** (SCREAMING_SNAKE_CASE)

- 

  示例: `MAX_RETRY_COUNT`, `DEFAULT_TIMEOUT`

### 5. 私有成员

- 

  以下划线 `_`开头

- 

  示例: `_privateMethod()`, `_internalState`

## 三、代码风格规范

### 1. 导入顺序

```dart
// 1. Dart核心库
import 'dart:async';
import 'dart:io';

// 2. Flutter库
import 'package:flutter/material.dart';

// 3. 第三方包
import 'package:dio/dio.dart';

// 4. 项目内部文件
import 'package:my_app/core/utils/logger.dart';
import 'package:my_app/presentation/widgets/custom_button.dart';
```

### 2. 缩进与空格

- 

  使用 **2个空格** 缩进

- 

  运算符前后加空格

```dart
// ✅ 正确
int sum = a + b;
if (condition) {
  doSomething();
}

// ❌ 错误
int sum=a+b;
if(condition){
doSomething();
}
```

### 3. 行长度

- 

  每行不超过 **120个字符**

- 

  超长应换行

### 4. 字符串

- 

  优先使用单引号 `'`

- 

  多行字符串使用三引号 `'''`或 `"""`

## 四、Widget 编写规范

### 1. 无状态组件

```dart
class UserAvatar extends StatelessWidget {
  final String imageUrl;
  final double size;

  const UserAvatar({
    super.key,
    required this.imageUrl,

        fit: BoxFit.cover,

```

### 2. 有状态组件

```dart
class CounterPage extends StatefulWidget {
  const CounterPage({super.key});

  @override
  State<CounterPage> createState() => _CounterPageState();
}

class _CounterPageState extends State<CounterPage> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: Text('Count: $_count'),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _increment,
        child: const Icon(Icons.add),
      ),
    );
  }
}
```

### 3. Widget 拆分原则

- 

  单个 Widget 不应超过 **300行**

- 

  复杂 UI 拆分为小组件

- 

  提取重复使用的 Widget

## 五、状态管理规范

### 推荐使用 Provider/Riverpod/Bloc

#### Provider 示例:

```dart
// Provider定义
final userProvider = ChangeNotifierProvider<UserNotifier>((ref) {
  return UserNotifier();
});

// Notifier类
class UserNotifier extends ChangeNotifier {
  User? _user;
  
  User? get user => _user;
  
  Future<void> fetchUser(String id) async {
    try {
      _user = await UserService.getUser(id);
      notifyListeners();
    } catch (e) {
      // 错误处理
    }
  }
}

// 页面中使用
class ProfilePage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final user = ref.watch(userProvider);
    
    return user == null 
        ? const CircularProgressIndicator()
        : Text(user.name);
  }
}
```

## 七、性能优化

### 1. 避免重建

```dart
// ✅ 使用 const 构造函数
const MyWidget()

// ✅ 使用 const 创建不可变对象
const SizedBox(height: 10)

// ❌ 避免不必要的重建
@override
void didUpdateWidget(covariant OldWidget oldWidget) {
  super.didUpdateWidget(oldWidget);
  // 避免在此处触发setState
}
```

### 2. 列表优化

```dart
// ✅ 使用 ListView.builder
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) {
    return ListTile(title: Text(items[index]));
  },
)

// ❌ 避免直接构建大列表
Column(children: items.map((e) => ListTile(...)).toList())
```

### 3. 图片缓存

```dart
// 使用 cached_network_image
CachedNetworkImage(
  imageUrl: url,
  placeholder: (context, url) => CircularProgressIndicator(),
  errorWidget: (context, url, error) => Icon(Icons.error),
)
```

### 2. API 异常处理

```dart
Future<User> getUser(String id) async {
  try {
    final response = await dio.get('/users/$id');
    return User.fromJson(response.data);
  } on DioException catch (e) {
    if (e.response?.statusCode == 404) {
      throw UserNotFoundException();
    }
    throw NetworkException();
  } catch (e) {
    throw UnknownException();
  }
}
```
